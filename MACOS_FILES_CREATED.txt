EXECUTABLE SCRIPTS
==================

1. setup-macos.sh
   - Purpose: One-time setup of development environment
   - What it does:
     * Checks for Xcode Command Line Tools
     * Installs Homebrew if needed
     * Installs SDL 1.2 and SDL_mixer
     * Verifies all dependencies
   - Usage: ./setup-macos.sh
   - Run: Once before first build

2. build-macos.sh
   - Purpose: Complete build system (build + DMG creation)
   - What it does:
     * Verifies dependencies
     * Builds the macOS application bundle
     * Creates the .dmg installer file
     * Shows build artifacts and next steps
   - Usage: ./build-macos.sh
   - Run: After setup, for each build

3. create-dmg.sh
   - Purpose: Standalone DMG creation from existing bundle
   - What it does:
     * Creates a distributable .dmg file
     * Includes Applications folder link
     * Optimizes with UDZO compression
   - Usage: ./create-dmg.sh
   - Run: When you need to recreate the DMG

4. sign-and-notarize.sh
   - Purpose: Code signing and notarization helper
   - What it does:
     * Provides menu-driven signing options
     * Supports ad-hoc signing (no account needed)
     * Supports developer certificate signing
     * Checks signature status
     * Removes quarantine attributes
   - Usage: ./sign-and-notarize.sh
   - Run: When you need to sign the app for distribution

5. verify-build.sh
   - Purpose: Build verification and validation
   - What it does:
     * Checks bundle structure
     * Verifies all resources are present
     * Validates Info.plist
     * Checks file permissions
     * Verifies Mach-O binary
   - Usage: ./verify-build.sh
   - Run: After building to verify success


BUILD CONFIGURATION FILES
=========================

1. Makefile.macos
   - Purpose: Primary macOS build configuration
   - What it contains:
     * Build rules for compiling source
     * App bundle creation targets
     * Info.plist generation
     * Resource bundling
     * DMG creation
     * Installation targets
   - Usage: make -f Makefile.macos [target]
   - Available targets:
     * make (default: build app)
     * make dmg (create installer)
     * make install (install to /Applications)
     * make clean (remove build artifacts)

BUILD OUTPUTS (created after running scripts)
==============================================

1. IceBreaker.app
   - Type: macOS application bundle
   - Created by: make -f Makefile.macos or ./build-macos.sh
   - Contents:
     * IceBreaker.app/Contents/MacOS/icebreaker (executable)
     * IceBreaker.app/Contents/Resources/ (game data)
     * IceBreaker.app/Contents/Info.plist (bundle metadata)
   - Size: ~2-3 MB (executable) + resources

2. icebreaker-2.2.2.dmg
   - Type: macOS disk image (installer)
   - Created by: make -f Makefile.macos dmg or ./build-macos.sh
   - Contains:
     * IceBreaker.app
     * Applications folder link
     * Formatted for easy drag-and-drop installation
   - Size: ~1-2 MB (compressed)
   - Distribution: Ready to upload and share


DEPENDENCIES INSTALLED BY setup-macos.sh
=========================================

1. Xcode Command Line Tools
   - Provides: clang compiler, make, standard headers
   - Size: ~500 MB

2. Homebrew
   - Provides: Package management system
   - Size: ~100 MB

3. SDL 1.2 (via Homebrew)
   - Provides: Simple DirectMedia Layer library
   - Version: 1.2.15+
   - Size: ~2 MB

4. SDL_mixer (via Homebrew)
   - Provides: Audio mixing library for SDL
   - Version: 1.2.12+
   - Size: ~1 MB


WORKFLOW
========

Initial Setup (One Time):
    1. ./setup-macos.sh         (installs dependencies)
    2. Review README.macos       (understand the system)

Building:
    1. ./build-macos.sh         (builds everything)
       OR
       make -f Makefile.macos   (just build app)
       make -f Makefile.macos dmg (create DMG)

Testing:
    1. ./verify-build.sh        (verify the build)
    2. open IceBreaker.app      (test the game)

Distribution:
    1. ./sign-and-notarize.sh   (optional signing)
    2. Share icebreaker-2.2.2.dmg with users
    3. Users drag IceBreaker.app to their Applications
